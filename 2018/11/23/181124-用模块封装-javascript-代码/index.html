用模块封装 JavaScript 代码

ES6 引入了模块化开发，为 JavaScript 构建大型项目提供了方便，解决了共享全局作用域的问题，并让 JavaScript 应用变得更有条理。

在 ES6 模块出现之前，社区就已经出现了模块化加载方案，最重要的有用于服务器端的 CommonJS 和用于浏览器端的 AMD、UMD，在此不做过多讨论。可以说，在广泛社区中，模块模式驱动了大部分代码，本文就 ES6 模块做简单分析。

何为模块

ES6 之前，我们通过 <script> 来加载 JavaScript 脚本，模块采用了不同方式来加载 JavaScript 脚本。ES6 使用基于文件的模块，通过模块的公开 API 暴露出模块中的方法属性。模块有如下特点：

  • 自动运行在严格模式下。
  • 顶级作用域的 this 值为 undefined。
  • 模块的顶级作用域即为模块本身。
  • 模块的 API 是静态的，即定义后无法从模块外部修改 API 功能。

模块的导出与导入

模块的最主要的优势就是导出与导入代码的能力，而不用将所有的代码都放在同一个文件中。ES6 提供了 export 和 import 关键字来导入导出模块。

1. 基本的导出

最基本的用法就是将 export 放在任意变量、函数或类之前，从模块中导出它们。创建一个称为 foo.js 的文件。

// foo.js

// 导出变量
export const number = 1

// 导出函数
export const sum = (num1, num2) => num1 + num2

// 导出类
export class Name {
  constructor(firstName, lastName) {
    this.firstName = firstName
    this.lastName = lastName
  }
}

// 稍后导出
const multiply = (num1, num2) => num1 * num2

export { multiply }

可以看出，除了 export 之外，每个声明都与正常形式完全一样。

2. 基本的导入

使用 import 可以访问被导出的模块中的功能。同目录下创建一个称为 bar.js 的文件。

// bar.js

// 导入 foo.js 的方法
import { number, sum, Name, multiply } from './foo.js'
console.log(sum(1, number)) // 2

ES6 模块哲学是按需导入，即用什么方法就导入什么方法。当然也可以把整个模块中的方法导入到一个单独的命名空间，然后通过命名空间访问。

// bar.js
……

// 导入 foo.js 的单个方法
import { number } from './foo.js'
console.log(number) // 1


// 完全导入 foo.js 模块
import * as Foo from './foo.js'
console.log(Foo.multiply(4, 5)) // 20

3. 重命名导出与导入

可以用 as 关键字来更改导入导出的名称。

// foo2.js
const sum = (num1, num2) => num1 + num2

export { sum as add }

导入方法。

// bar2.js
import { add as addTwo } from './foo2.js'
console.log(addTwo(1, 1))

4. 默认导出与导入

可以通过 default 关键字来指定模块中默认导出的方法。

第一种方法，无需指定方法名，因为方法即代表模块本身。

// sum.js
export default (num1, num2) => num1 + num2

导入方法。

// bar.js
import sum from './sum.js'
console.log(sum(1, 2)) // 2

第二种方法，类似常规导出，增加 default 关键字。导入也是按常规方法导入即可

// header.jsx

const Header = (props) => <h1>{props.children}</h1>

export default Header

如果同时用默认导出和普通导出，可以用如下方法。

// sample.js

export const number = 1

export default (num1, num2) => num1 + num2

导入方法。

import sample, { number } from './sample.js'

// 或者
import { default as sum, number } from './sample.js'

需要注意的是，默认名称必须位于非默认名称之前，非默认名称依旧要用花括号包裹起来。

在浏览器中使用模块

使用方法很简单，只要将 script 标签中的 type 类型改为 module 即可，即 type="module"。需要注意的是浏览器对模块的加载都是异步的，等同于添加了 defer 关键字。

<script type="moudle" src="./module.js"></script>

// 等同于
<script type="moudle" src="./module.js" defer></script>

也可以使用内联方法。

<script type="moudle">
import { sum } from './sample.js'

let res = sum(1, 1)
</script>
