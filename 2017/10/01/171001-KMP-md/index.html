KMP算法是搜索功能常用的一种算法。其本质就是字符串匹配方法的一种改进。

引言

字串符匹配是计算机的基本任务之一，说白了，有一个很单纯的想法: 对一个字符串，从左到右一个个匹配，遇到不匹配的就跳回去，并向后推进一个重新匹配，直到完全匹配或匹配结束为止。这种想法实现起来很容易，有一个问题是会做大量的无用功，后面我们会提到。KMP 算法可以说是这种想法的一个改进版。

KMP 算法

1. 问题引入

有一个字符串“ABC ABCDAB ABCDABCDABDE”，我想知道里面是否包含了“ABCDABD”这个字符串，应该怎么去做呢？

我们称字符串“ABC ABCDAB ABCDABCDABDE”为文本串 S ，字符串“ABCDABD”为模式串 P 。并假设现在文本串 S 匹配到 i 位置，模式串 P 匹配到 j 位置。

2. 思路分析

               1         2  
   i 01234567890123456789012
   S ABC ABCDAB ABCDABCDABDE
   P ABCDABD
   j 0123456

（1）模式串 P 的第一个字母 A 与文本串 S 的第一个字母 A 进行比较，匹配，称作 S[0] = P[0] ,依次对比得 S[1] = P[1] , S[2] = P[2] ,发现 S[3] ！= P[3] 。接着有两种选择，一是将 P 向后移动一位，重新匹配；二是根据前面的信息移动不止一位，略过一些字符串。

   第一种选择：
               1         2  
   i 01234567890123456789012
   S ABC ABCDAB ABCDABCDABDE
   P  ABCDABD
   j  0123456

   第二种选择：
               1         2  
   i 01234567890123456789012
   S ABC ABCDAB ABCDABCDABDE
   P    ABCDABD
   j    0123456

（2）由第（1）步我们已经知道 S[1] ~ S[3] 不与 P[0] 相等，因此选择第二种模式会减少匹配次数。

               1         2  
   i 01234567890123456789012
   S ABC ABCDAB ABCDABCDABDE
   P     ABCDABD
   j     0123456

（3）我们再往后移动一位发现 S[10] 与 P[6] 有点差异，并且 AB 在字符串的头尾出现了两次，这意味着下次匹配的起点应该为尾端的 AB ，因此令 i = 8，j = 2 ，继续匹配。

               1         2  
   i 01234567890123456789012
   S ABC ABCDAB ABCDABCDABDE
   P         ABCDABD
   j         0123456

（4）在 i = 10 处又不相等，继续向后移动，令 i = 10，j = 0，继续匹配。

               1         2  
   i 01234567890123456789012
   S ABC ABCDAB ABCDABCDABDE
   P           ABCDABD
   j           0123456

               1         2  
   i 01234567890123456789012
   S ABC ABCDAB ABCDABCDABDE
   P            ABCDABD
   j            0123456

（5）一直向后匹配，到上图这里发现又出现了和第（3）步类似的情况，同样的，令 i = 15，j = 2 ，继续匹配。

               1         2  
   i 01234567890123456789012
   S ABC ABCDAB ABCDABCDABDE
   P                ABCDABD
   j                0123456

现在我们找到完全匹配的字符串了，其起始位置在 S[15] 处。

3. 部分匹配表

上面的匹配模式虽然减少了匹配次数，但也带来了一个问题，就是我们怎么量化向后移动的位数呢？为此我们引入了一个 next 数组来解决问题，在这之前，我们先来寻找一下前后缀最长公共元素的长度来完成我们的部分匹配表。

TIPS：
前后缀：不包含最后一个或第一个字符的其他字符的组合。比如说对字符串 hope 而言，其前缀有 h，ho，hop；后缀有 ope，pe，e 。
部分匹配值： 前缀和后缀的最长共有元素的长度。

模式串 P A B C D A B D
部分匹配值 0 0 0 0 1 2 0

以”ABCDABD”为例，

字符串 前缀 后缀 共有元素长度
A 无 无 0
AB [A] [B] 0
ABC [A, AB] [BC, C] 0
ABCD [A, AB, ABC] [BCD, CD, D] 0
ABCDA [A, AB, ABC, ABCD] [BCDA, CDA, DA, A] A 1
ABCDAB [A, AB, ABC, ABCD, ABCDA] [BCDAB, CDAB, DAB, AB, B] AB 2
ABCDABD [A, AB, ABC, ABCD, ABCDA, ABCDAB] [BCDABD, CDABD, DABD, ABD, BD, D] 0

部分匹配的实质是什么呢？当子串头部和尾部有重复的时候，搜索移动的位数和部分匹配值就挂钩了。根据上面的表格，我们可以得到一个公式来解决移动位数的问题。

$$ 移动位数 = 已匹配的字符数 - 对应部分的部分匹配值 $$

举例来说，“ABCDAB”中首尾各有 “AB”，部分匹配值为 2 ，那么它移动位数就为 6 - 2 = 4 ，即搜索的时候只要向后移动 4 位就可以到达下一个“AB”的位置，这和我们前面的操作是相匹配的。

4. next 数组

next 数组的公式如下，

$$ next[j]= \begin{cases} -1 & (j=0) \ max | k: 0<k<j,P[0,k-1]=P[j-k,j-1] \ 0 &(其他情况)\end{cases} $$

next 数组考虑的是除当前字符外的最长相同前后缀。

  • 初始值即 next[0] 赋值为 0 。
  • next[j] = k 代表p[j] 之前的模式串子串中，有长度为k 的相同前缀和后缀。
    举例来说，对于字串符“ABCDAB”来说，第 6 个字符 B 之前的字符串“ABCDA”有长度为 1 的相同前后缀，所以第 6 个字符对应的 next 值为 1；再比如说对于字串符“ABCDABD”来说，第 7 个字符 D 之前的字符串“ABCDAB”有长度为 2 的相同前后缀，所以第 7 个字符对应的 next 值为 2 。

则由上面的推导我们可以得到 next 数组的表格，

模式串 P A B C D A B D
部分匹配值 0 0 0 0 1 2 0
next 数组 -1 0 0 0 0 1 2

对比部分匹配值和 next 数组我们发现其实只要将部分匹配值整体向右移动一位，然后初始值赋为 -1 即可得到 next 数组对应的值。
同样的，我们也可以得到一个公式来计算向右移动的位数，

$$ 移动位数 = 失配字符所在的位置 - 失配字符对应的 next 值 $$

用数学表达是表示就是

$$ 移动位数 = j - next[j] $$

举例来说，如下图所示，字符串“ABCDABD”的失配字符 D 对应的 j = 6，相应的 next 值为 2 ，那么它移动位数就为 6 - 2 = 4 ，即搜索的时候只要向后移动 4 位就可以到达下一个“AB”的位置。

               1         2  
   i 01234567890123456789012
   S ABC ABCDAB ABCDABCDABDE
   P     ABCDABD
   j     0123456

看到这里，我们应该可以惊喜的发现，基于“部分匹配值表”和基于“next 数组”得来的结果是一样的。对比两个公式我们可以很轻松的发现

                     移动位数 = 已匹配的字符数 - 对应部分的部分匹配值 
                     移动位数 = 失配字符所在的位置 - 失配字符对应的 next 值

                     失配字符对应的 next 值 = 对应部分的部分匹配值 + 1
                  => 失配字符所在的位置 = 已匹配的字符数 + 1

因此结果必然是相同的，所以其实将“部分匹配值表”当做“next 数组”来用也是可以的。

5. 代码实现

//KMP 算法
int KMPSearch(char *s,char *p)
{
  int sLen = strlen(s);
  int plen = strlen(p);
  int i = 0;
  int j = 0;
  //引入 next 数组
  GetNext(char *p, int next[]);

  while(i < sLen && j < pLen)
  {
    //当 j = -1 或者匹配成功时，都令 i++,j++
    if( j == -1 || s[i] == p[j])
    {
      i++;
      j++;
    }
    else
    {
      //当 j != -1 且匹配失败时，令 i 不变，j 回退 next[j]
      j = next[j];
    }
  }
  //若匹配成功，则返回字符串的位置
  if(j == pLen)
    return i - j;
  //若失败，则返回 -1
  else
    return -1;
}

//next 数组
void GetNext(char *p, int next[])
{
  int pLen = strlen()
  next[0] = -1;
  int k = -1;
  int j = 0;

  while (j < pLen - 1)
  {
    //p[k]表示前缀，p[j]表示后缀
    if (k == -1 || p[j] ==p[k]
   {
     k++;
     j++;
     next[j] = k;
   }
   else
   {
     k = next[k];
   }
  }
}
