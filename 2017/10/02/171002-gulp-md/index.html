使用gulp实现自动化工作流

这篇文章是讲述如何使用自动化工具gulp来增强项目开发的工作流程的。只需简单几步就可以使我们的项目如鱼得水。

gulp是一个前端自动化构建工具，它可以帮助我们做以下事情：

  • 压缩文件(图片、css文件，js文件…)
  • 合并文件(雪碧图、html、css、js…)
  • 混淆代码
  • 部署项目到远程服务器
  • ……

总之，gulp非常方便多人协作开发，它可以帮助我们高效简洁地开发项目。

搭建gulp自动化工作流大概需要做这么几件事情。

  • 安装gulp
  • 创建项目目录树
  • 配置gulp
  • 使用gulp开发

下面让我们一起开启gulp使用之旅。

安装gulp

由于gulp是基于node.js的，因此我们先应该安装node.js。如果嫌npm太慢，万能的淘宝已经给我们打包好了镜像，只需要输入以下命令即可享受飞速安装的快感。

  1. 全局安装gulp

  2. 在项目根目录下安装gulp，会有node_module文件夹和package.json文件出现。

  3. 在项目根目录下创建名为gulpfile.js的文件，这个文件将用来管理我们的任务。

到这里，我们的gulp环境就建好了，算是拿到了开启宝藏的第一把钥匙。然我们继续旅行。

创建项目目录树

以最常见的网站来解说，包含了banner，nav，slider，content和footer，我们可以构建出如下的目录树。

  1. dist 为最终输出的文件
  2. doc 保存各种文档，方便查询

  3. html 存放HTML文件

    • index.html 页面输出
    • pages 布局模板及内容页
    • templates 组件

  4. images 存放图片文件

    • favicon 存放网页图标
    • icon 存放要制成雪碧图的零散图片
    • icon-2x 存放供移动端使用的二倍图
    • 其余图片可直接放在根目录下

  5. js 存放js文件

    • jquery.js 保存引入的jQuery
    • main.js 保存原生js

  6. scss 存放sass文件

    • main.scss 保存宿主文件，最后输出可直接链到这里
    • init 初始化文件
      • _init.scss 引用该目录下的其他sass文件，这样在main中就只需要链到这里
      • _common.scss 保存一些共同样式
      • _reset.scss 浏览器默认样式
      • _sprite.scss 雪碧图
      • _sprite@2x.scss 二倍图
    • var 变量文件
      • _var.scss 引用该目录下的其他sass文件
      • _color.scss 颜色变量
      • _dimen.scss 基本尺寸
      • _font-family.scss 字体变量
      • _font-szie.scss 字体大小变量
    • widget 存放单独组建的样式
      • _main.scss 引用该目录下的其他sass文件
      • 分别对应各个组件的样式

第二把钥匙到手，可以开启通向下一站的大门了。下一站，配置站。
