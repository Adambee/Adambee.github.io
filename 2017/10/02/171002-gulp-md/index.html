使用gulp实现自动化工作流

这篇文章是讲述如何使用自动化工具gulp来增强项目开发的工作流程的。只需简单几步就可以使我们的项目如鱼得水。

gulp是一个前端自动化构建工具，它可以帮助我们做以下事情：

  • 压缩文件(图片、css文件，js文件…)
  • 合并文件(雪碧图、html、css、js…)
  • 混淆代码
  • 部署项目到远程服务器
  • ……

总之，gulp非常方便多人协作开发，它可以帮助我们高效简洁地开发项目。

搭建gulp自动化工作流大概需要做这么几件事情。

  • 安装gulp
  • 创建项目目录树
  • 配置gulp
  • 使用gulp开发

下面让我们一起开启gulp使用之旅。

安装gulp

由于gulp是基于node.js的，因此我们先应该安装node.js。如果嫌npm太慢，万能的淘宝已经给我们打包好了镜像，只需要输入以下命令即可享受飞速安装的快感。

    $ npm install -g cnpm --registry=https://registry.npm.taobao.org
  1. 全局安装gulp
     $ cnpm install gulp
  2. 在项目根目录下安装gulp，会有node_module文件夹和package.json文件出现。
     $ cnpm install gulp --save-dev
  3. 在项目根目录下创建名为gulpfile.js的文件，这个文件将用来管理我们的任务。
     $ touch gulpfile.js

到这里，我们的gulp环境就建好了，算是拿到了开启宝藏的第一把钥匙。然我们继续旅行。

创建项目目录树

以最常见的网站来解说，包含了banner，nav，slider，content和footer，我们可以构建出如下的目录树。

  1. dist 为最终输出的文件
  2. doc 保存各种文档，方便查询

  3. html 存放HTML文件

    • index.html 页面输出
    • pages 布局模板及内容页
    • templates 组件

  4. images 存放图片文件

    • favicon 存放网页图标
    • icon 存放要制成雪碧图的零散图片
    • icon-2x 存放供移动端使用的二倍图
    • 其余图片可直接放在根目录下

  5. js 存放js文件

    • jquery.js 保存引入的jQuery
    • main.js 保存原生js

  6. scss 存放sass文件

    • main.scss 保存宿主文件，最后输出可直接链到这里
    • init 初始化文件
      • _init.scss 引用该目录下的其他sass文件，这样在main中就只需要链到这里
      • _common.scss 保存一些共同样式
      • _reset.scss 浏览器默认样式
      • _sprite.scss 雪碧图
      • _sprite@2x.scss 二倍图
    • var 变量文件
      • _var.scss 引用该目录下的其他sass文件
      • _color.scss 颜色变量
      • _dimen.scss 基本尺寸
      • _font-family.scss 字体变量
      • _font-szie.scss 字体大小变量
    • widget 存放单独组建的样式
      • _main.scss 引用该目录下的其他sass文件
      • 分别对应各个组件的样式

第二把钥匙到手，可以开启通向下一站的大门了。下一站，配置站。

配置gulp

先看看我们的配置文件gulpfile.js长什么样儿。

//引入gulp工具
var gulp = require('gulp');
//载入gulp模块
fileInclude = require('gulp-file-include'),
sass = require('gulp-sass'),
autoprefixer = require('gulp-autoprefixer'),
minifycss = require('gulp-minify-css'),
jshint = require('gulp-jshint'),
uglify = require('gulp-uglify'),
imagemin = require('gulp-imagemin'),
rename = require('gulp-rename'),
clean = require('gulp-clean'),
concat = require('gulp-concat'),
notify = require('gulp-notify'),
cache = require('gulp-cache'),
livereload = require('gulp-livereload')
browserSync = require('browser-sync'); 

//html文件
gulp.task('html',function() {
  return gulp.src('html/*.html')
  .pipe(fileInclude({ prefix: '@@', basepath: '@file' }))
  .pipe(gulp.dest('dist'))
  .pipe(notify({ message: 'Html tasks complete.' })); //返回成功提示
});

//样式
gulp.task('style',function() {
  return gulp.src('scss/main.scss')
    .pipe(sass({ style:'expanded' }))  //编译sass
    .pipe(autoprefixer('last 2 version','safari 5', 'ie 8', 'ie 9', 'opera 12.1', 'ios 6', 'android 4'))  //添加浏览器前缀
    .pipe(gulp.dest('dist/css'))
    .pipe(rename({suffix: '.min'})) //重命名
    .pipe(minifycss()) //压缩
    .pipe(gulp.dest('dist/css')) //输出压缩文件
    .pipe(notify({ message: 'Styles task complete.' }));
});

//脚本
gulp.task('script', function () { 
  return gulp.src('js/*.js')  //匹配所有js目录下的js文件
    .pipe(jshint()) //检测代码错误和潜在问题
    .pipe(jshint.reporter('default'))
    .pipe(concat('main.js'))  //合并所有js到main.js中
    .pipe(gulp.dest('dist/js')) //保存在dist/js文件夹下
    .pipe(rename({suffix: '.min'})) //重命名
    .pipe(uglify()) //压缩代码
    .pipe(gulp.dest('dist/js')) //保存压缩文件在dist/js下面
    .pipe(notify({ message: 'Scripts task complete.' })); //返回成功提示
});

//图片压缩
gulp.task('image',function() {
  return gulp.src('images/**/*')
    .pipe(cache(imagemin({ optimizationLevel: 5, progressive: true, interlaced: true })))
    .pipe(gulp.dest('dist/images'))
    .pipe(notify({ message: 'Images task complete.' }));
});

//清除目录重建文件
gulp.task('clean', function() {  
  return gulp.src(['dist/*.html','dist/css', 'dist/js', 'dist/images'], {read: false})
    .pipe(clean());
});

//建立预设任务，只输入gulp时执行
gulp.task('default',function() {  
    gulp.start('html','style', 'script', 'image');
});

//监视所有文件
gulp.task('watch',function() {
  gulp.watch('dist/*.html',['html']);
  gulp.watch('dist/css/**/*',['style']);
  gulp.watch('dist/js/*.js',['script']);
  gulp.watch('dist/images/**/*',['image']);
});

//即时重整
gulp.task('watch', function() {
  var server = livereload();
  gulp.watch(['dist/**']).on('change', function(file) {
    server.changed(file.path);
  });
});

//浏览器同步
gulp.task('sync', function () { 
  var files = [ 
    'dist/*.html', 
    'dist/css/*.css', 
    'dist/images/*.png', 
    'dist/js/*.js' 
  ];
  browserSync.init(files, { 
    server: { 
      baseDir: './dist' 
    } 
  }); 
});

下面我们来详细解读gulp的工作原理，gulp是通过steam流来实现整个流程的。说白了就是读取=>操作=>输出 的模式。
仔细搞懂下面这段代码，我们就可以掌握gulp的精髓所在

    var gulp = require('gulp');
    var uglify = require('gulp-uglify')；
    gulp.task('script', function () { 
        return gulp.src('js/*.js') 
        .pipe(concat('main.js')) 
        .pipe(rename({suffix: '.min'}))
        .pipe(uglify()) 
        .pipe(gulp.dest('dist')); 
    });
  • 前两行分别载入gulp模块和用来压缩js的gulp-uglify模块。
  • gulp.task()是一个gulp的API用来建立任务，这里建立了一个名为script的任务。
  • gulp.src()用字符串匹配一个文件，这里匹配了js目录下的所有以.js结尾的文件。
  • .pipe(concat())将所有js文件整合成一个文件。
  • .pipe(rename({suffix: '.min'}))用来重命名文件。
  • .pipe(uglify())用来压缩js。
  • .pipe(gulp.dest())输出文件，本例中将输出名为main.js和main.min.js的文件在dist目录下。

看到这里我们应该了解了gulp的工作原理了，只要再结合上面的配置文件，就可以轻松搞定最为艰难的一关了。第四把钥匙到手了，这扇门的背后，将会是一个全新的世界。

使用gulp开发

和其他应用使用起来完全相同，只需要在根目录下输入

     $ gulp <task> <othertask>

例如只要键入gulp script就可以执行前面我们的任务了。

到这里，我们的gulp之旅就算结束了，接下来便可以开始愉快的用gulp开发了。
