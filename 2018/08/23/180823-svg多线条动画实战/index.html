我们需要做的就是通过操作 path 元素来实现动画效果。svg 路径动画的原理其实就是通过改变线条的偏移量来欺骗视网膜从而实现动画效果，这里需要用到两个属性，简单提一下。

  1. stroke-dasharry: <dasharray> 指定短划线和缺口的长度。
  2. stroke-dashoffset 指定了 dash 模式到路径开始的距离，其实就是线条的偏移量。

单个线条动画用 animation 可以简单实现，拿中间的小企鹅为例，两分钟实现动画。

#path3 {
  animation: move3 2s ease-in-out infinite;
  stroke-dasharray: 384.932861328125px;
}

@keyframes move3 {
  0% {
    stroke-dashoffset: 384.932861328125px;
  }
  100% {
    stroke-dashoffset: 0;
  }
}

下面是实现后的效果，看起来还挺不错的。

基本的思路就是指定短划线长度为整个线条的长度，先让线条完全偏移，在一段时间内改变偏移量，让线条逐渐出现。而我们可以通过下面这行代码来获取线条长度。

document.getElementById('path3').getTotalLength()

2. 多个线条依次出现动画

最开始的的思路，给每个线条加动画，然后依次给线条延时，循环往复，从而实现依次出现的视觉效果。一点都难不倒我们。

#path1 {
  animation: move1 2s ease-in-out infinite;
  stroke-dasharray: 591.3617553710938px;
}
#path2 {
  animation: move2 1s ease-in-out 2s infinite;
  stroke-dasharray: 141.3917236328125px;
}
#path3 {
  animation: move3 2s ease-in-out 3s infinite;
  stroke-dasharray: 384.932861328125px;
}

@keyframes move1 {
  0% {
    stroke-dashoffset: 591.3617553710938px;
  }
  100% {
    stroke-dashoffset: 0;
  }
}
@keyframes move2 {
  0% {
    stroke-dashoffset: 141.3917236328125px;
  }
  100% {
    stroke-dashoffset: 0;
  }
}
@keyframes move3 {
  0% {
    stroke-dashoffset: 384.932861328125px;
  }
  100% {
    stroke-dashoffset: 0;
  }
}

然而，实现后的效果却长这样。忍不住念了两句诗，多情自古空余恨，好梦由来最易醒。

赶紧分析原因，发现就是“循环往复”这儿出了问题，各个线条各干各的，只是在第一次播放的时候有延时，以后就会争先恐后地跑出来，没有相互关联起来，我们设置延时的初衷没有达到啊。

现在我们是将动画分成了三个，才会出现上面的情况，知道了症结所在，就可以对症下药了。如果我们将他们合为一个，并将整个动画分割成三段，每段控制一个线条，岂不是就可以完美解决延时问题了？画张图来分析一下。

于是我们就可以很轻松写出三段动画了，首先分析每段线条动画所占的时间比例，它只在比例限定的时间里活跃，其他时间处于静止状态，这样我们也不需要设置延时了，我们将三段动画的时间比例分别定为 40%，20%，40%，来看看代码长什么样。

#path1 {
  animation: move1 5s ease-in-out infinite;
  stroke-dasharray: 591.3617553710938px;
}
#path2 {
  animation: move2 5s ease-in-out infinite;
  stroke-dasharray: 141.3917236328125px;
}
#path3 {
  animation: move3 5s ease-in-out infinite;
  stroke-dasharray: 384.932861328125px;
}

@keyframes move1 {
  0% {
    stroke-dashoffset: 591.3617553710938px;
  }
  40%,
  100% {
    stroke-dashoffset: 0;
  }
}
@keyframes move2 {
  0%,
  40% {
    stroke-dashoffset: 141.3917236328125px;
  }
  60%,
  100% {
    stroke-dashoffset: 0;
  }
}
@keyframes move3 {
  0%,
  60% {
    stroke-dashoffset: 384.932861328125px;
  }
  100% {
    stroke-dashoffset: 0;
  }
}

现在，想要的效果出现了，这就是柳暗花明又一村最佳表现啊。

依次出现的动画到此告一段落。

3. 多线条依次消失动画

有了上面的经验，消失动画就变得特别简单了，只需要将每个线条的消失动画也加进总动画即可。同样的，画个图来分析一下。

需要注意的是，50%~64% 和 86%~100% 两段时间内是没有动画的。看一下详细代码。

@all-time: 8s;

#path1 {
  animation: move1 @all-time ease-in-out infinite;
  stroke-dasharray: 591.3617553710938px;
}

#path2 {
  animation: move2 @all-time ease-in-out infinite;
  stroke-dasharray: 141.3917236328125px;
}

#path3 {
  animation: move3 @all-time ease-in-out infinite;
  stroke-dasharray: 384.932861328125px;
}

@keyframes move1 {
  0% {
    stroke-dashoffset: 591.3617553710938px;
  }
  22%,
  64% {
    stroke-dashoffset: 0;
  }
  72%,
  100% {
    stroke-dashoffset: -591.3617553710938px;
  }
}

@keyframes move2 {
  0%,
  22% {
    stroke-dashoffset: 141.3917236328125px;
  }
  32%,
  72% {
    stroke-dashoffset: 0;
  }
  78%,
  100% {
    stroke-dashoffset: -141.3917236328125px;
  }
}

@keyframes move3 {
  0%,
  32% {
    stroke-dashoffset: 384.932861328125px;
  }
  50%,
  78% {
    stroke-dashoffset: 0;
  }
  86%,
  100% {
    stroke-dashoffset: -384.932861328125px;
  }
}

最终实现后的效果如开篇所示，并且发现只需要控制总时间就可以控制整个动画的速度，这才是我们想要的。完整的代码可看下面👇

总结一句，个人觉得 svg 线条动画关键点就是多个线条的整体控制，只要解决了这个问题，整个流程就比较简单了。
